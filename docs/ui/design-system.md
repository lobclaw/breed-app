## 十、使用规则

1. **所有颜色必须通过 CSS 变量引用**，禁止硬编码 hex 值（icon 背景除外，因为需要比 soft 深一档的独立值）
2. **阴影必须用粉调**（rgba(234,62,119,x)），不用灰色阴影
3. **暗色模式的背景用暖棕黑**（#181210），不用冷蓝黑
4. **功能色在暗色模式下提亮**，保持可读性
5. **左边框用 border-left 实现**（跟随卡片圆角），不用 ::before 伪元素画
6. **渐变色晕用 ::before 伪元素**，卡片内容用 `> *` 选择器设 z-index: 1 浮在上面
7. **所有交互用 CSS transition**，不用 JS 动画（checkbox 切换除外）

## 十一、加载与错误状态

### 骨架屏（Loading Skeleton）

所有列表页和详情页在数据加载时显示骨架屏：
- 卡片骨架：白色卡片底 + 内部灰色圆角条块
- 标题位：宽60%, 高14px, 圆角4px, 背景 --card-dim
- 副标题位：宽40%, 高12px
- 按钮位：宽30%, 高32px
- 动画：左到右渐变光扫过（shimmer），周期1.5s
- 骨架颜色：--card-dim（#FFF0E8）
- 高光色：白色20%透明度

### 错误状态（3种）

**网络错误：**
- 居中布局：WiFi 图标（48px, --text-3）+ "网络连接失败"（16px/700, --text-1）+ "请检查网络后重试"（13px, --text-2）+ "重试"按钮（--primary ghost）

**加载失败：**
- 居中布局：⚠️ 图标（48px）+ "加载失败"（16px/700）+ "点击重试"（--primary 文字链接）

**权限不足（协助者）：**
- 居中布局：🔒 图标（48px）+ "无权访问"（16px/700）+ "此功能仅管理员可用"（13px, --text-2）+ "返回"按钮（ghost）

### 表单必填规范

- **不使用红星（*）标记必填字段**
- 必填字段：无任何标记（默认即必填）
- 选填字段：label 后加灰色小字"（选填）"，使用 --text-3 颜色，13px
- 提交校验失败时：未填的必填字段边框变为 --red，下方显示 "请填写此项"（12px, --red）

## 十二、表单智能预填规则

### 犬只预填（所有包含犬只选择器的表单统一遵守）

**优先级 1：上下文预填**
从犬只详情页或首页任务卡片进入表单 → 直接预填该犬只

**优先级 2：按表单类型智能推荐**
从 FAB Action Sheet 或 R-1 快速记录进入时，根据表单类型推荐最可能的犬只：

| 表单类型 | 智能推荐策略 |
|---------|------------|
| 临产监测 | 距预产期最近的犬 |
| 发情记录 | 最近有发情前兆标记的犬 |
| 配种记录 | 当前处于发情中的犬 |
| 孕检/产检 | 当前怀孕中的犬 |
| 疫苗/驱虫 | 上次同类型记录的犬（批量操作时预选上次的批次） |
| 用药任务 | 当前生病中/用药中的犬 |
| 支出录入 | 不推荐犬只（大部分支出不关联犬只） |
| 体重记录 | 上次记录体重的窝 |

**优先级 3：无匹配**
以上都不匹配时，显示犬只选择器，不预填。

### 预填 UI 规范

预填后犬只显示为信息卡片（头像 + 名字 + 品种 + 推荐原因），下方有「更换犬只 →」文字链接。用户始终可以更换，预填不是强制。

```
┌─────────────────────────────────────┐
│ 🐩 花花                        ✓   │ ← 已预填
│ 马尔济斯 · 种母 · 距预产期3天       │ ← 推荐原因
└─────────────────────────────────────┘
更换犬只 →
```

未预填时显示空的犬只选择器卡片：
```
┌─────────────────────────────────────┐
│ 🐾 点击选择犬只                  >  │
└─────────────────────────────────────┘
```

### 其他输入记忆

- **药品名称**（疫苗类型/驱虫药品）：记忆最近使用的 5 种，下次自动置顶展示
- **日期**：默认今天，日期选择器下方提供「今天/昨天/前天」快捷 chip
- **支出分类**：记忆最近使用的分类，在分类选择 Sheet 的「最近使用」区域展示

## 十三、快速监测日志模式

> 适用于「高频重复录入」场景：同一只犬在短时间内多次记录同类数据。与普通一次性表单的区别是：显示历史对比、征兆勾选网格、自动记录时间。

### 适用场景

| 场景 | 核心数据 | 勾选征兆 |
|------|---------|---------|
| 临产监测（R-7） | 体温 | 刨窝/焦躁/食欲减退/喘气/分泌物/宫缩/乳汁/拒食 |
| 发情观察（R-19） | 外阴状态 | 桃子硬/桃子软/接受爬跨/出血量变化/行为异常 |
| 幼犬体重记录（D-19） | 体重(g) | — (无征兆，但有上次对比+趋势线) |

### 布局规范

```
┌──────────────────────────────────────┐
│ ← 录入[监测类型]                      │ 顶栏（通用标题，不含犬名）
│   [上下文信息：预产期/发情天数等]     │ 副标题，--text-3
│                                      │
│ ┌─ 犬只/窝次选择卡片 ─────────────┐  │
│ │ 🐾 [犬名/窝名]          ＞      │  │ 点击切换犬只/窝次
│ │    [品种 · 角色 · 关键状态]     │  │ 16px圆角，var(--shadow)
│ └────────────────────────────────┘  │ 带主题色渐变背景
│                                      │
│ ── 核心数值 ─────────────────────── │
│  [当前值] [单位]   上次: [值] · [时间] │ 对比上次
│  ⚠️ [智能提示：变化量/阈值]          │ 条件显示
│  📈 [迷你趋势线：最近4-6次]          │ 可选
│                                      │
│ ── 观察到的征兆（点击勾选）────────── │
```

### 核心组件规范

**犬只/窝次选择卡片（dog-select-card / litter-card）：**
- 与体重记录页的窝选择卡片统一样式
- 16px 圆角，padding 14px 16px，var(--shadow)
- 42px 圆形头像，渐变背景（按主题色），pets 图标白色
- 右侧 chevron_right 图标（--text-4），整卡可点击切换
- 渐变叠加层：`linear-gradient(135deg, 主题色-soft 0%, transparent 40%)`
- 顶栏标题使用通用名称（如「录入临产监测」），不含犬名
- 犬名显示在选择卡片中，meta 显示关键状态信息

**上次对比区域：**
- 当前值：28px/800，功能色（体温用 --text-1，异常时用 --red）
- 单位：14px/600，--text-3
- 「上次: 37.5°C · 4小时前」：12px/500，--text-3
- 变化提示：「⚠️ 比上次下降0.6°C」：13px/600，--amber 或 --red

**迷你趋势线：**
- 高度 40px，宽度 100%
- 用 SVG 绘制折线 + 圆点
- 线色：--primary，点色：--primary，末尾点高亮
- 背景区域填充：--primary-soft 半透明

**征兆勾选网格：**
- 2 列布局，gap 8px
- 每项高度：44px（大触控目标，脏手可操作）
- 未选：--card-dim 背景，14px 圆角，文字 14px/500 --text-2
- 已选：--primary-soft 背景，--primary 文字，左侧 ✓ 图标
- 点击切换，无需长按

**时间显示：**
- 自动记录当前时间（时:分），显示在保存按钮旁
- 可点击修改（打开时间选择器）
- 格式：HH:MM，14px/600，--text-3

### 与普通表单的区别

| | 普通表单 | 快速监测日志 |
|---|---|---|
| 犬只选择 | 需要手动选 | 智能预填（第十二节规则） |
| 时间记录 | 只有日期 | 日期 + 时间（自动） |
| 历史对比 | 无 | 显示上次值 + 变化量 + 趋势线 |
| 征兆记录 | toggle/segmented/textarea 混合 | 统一为勾选网格 |
| 文本框 | 可能有多个 | 只有一个「补充说明」 |
| 录入耗时 | ~30秒 | ~10秒 |

## 十四、红涨绿跌全局规则

> 全局统一中式惯例：红色=上升/增长/进账，绿色=下降/减少/出账。适用于所有涉及数值变化的场景。

| 场景 | 上升/增长 | 下降/减少 |
|------|---------|---------|
| 财务收支 | 收入 = --red（红） | 支出 = --green（绿） |
| 体重变化 | 增重 ↑ = --red（红） | 减重 ↓ = --green（绿） |
| 趋势线 SVG | 上升折线 = #e05252（红） | 下降折线 = #3dae6f（绿） |
| 数值对比 delta | +16g = --red | -6g = --green |
| 净利润/净收益 | 正值 = --primary（品牌色） | 负值 = --green（绿） |

**注意：不是所有红色都代表「好」，不是所有绿色都代表「坏」。** 红绿只表示「方向」（涨/跌），不表示「好坏」。例如幼犬体重下降用绿色，虽然下降是坏事，但颜色遵循方向规则保持一致性。
