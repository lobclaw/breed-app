# 当前路线图

## 当前目标

当前主线是在现有 UniApp + UniCloud 应用之上，继续推进首页工作台密度自适应优化，同时保持既有业务规则、事实源和交互语义稳定。

## 当前里程碑

`首页工作台密度自适应优化`

本轮原则：

- 保持首页固定四层 `逾期 / 繁育 / 健康 / 用药`
- 不恢复统一任务池、摘要卡、二级页或通用 Sheet
- 不新增首页后端事实源
- 在高密度场景下提升首页可读性与可操作性

## 固定阶段顺序

1. 契约 / 测试底座
2. 繁育步骤工作台
3. 健康批量优先工作台
4. 用药状态工作台
5. 逾期 / 计数 / 防闪回校准
6. 今日重点

## 各阶段目标

### 1. 契约 / 测试底座

- 建立稳定的首页 workbench 视图模型
- 补齐 adapter 与 task-service 的关键测试
- 保证首页副作用边界不被破坏

### 2. 繁育步骤工作台

- 繁育区按步骤组织
- 保持单犬/单窝推进语义
- 额外安排继续独立成层

### 3. 健康批量优先工作台

- 保留疫苗/驱虫的真实批量价值
- 部分完成时保持局部承接与元数据同步
- 疾病观察继续归属健康区

### 4. 用药状态工作台

- 用药区按 `未完成 / 部分完成 / 已完成` 组织
- 持续以 `medication_tasks` 为唯一事实源
- 首页与详情页共用同一套状态语义

### 5. 逾期 / 计数 / 防闪回校准

- pills 数字、WeekStrip 红点、首页可见内容一致
- latest token、suppression、乐观移除机制继续有效
- 高密度场景下不闪回、不重算错误

### 6. 今日重点

- 只在前五阶段稳定后再做
- 今日重点只是聚焦层，不是第二套任务系统

## 当前非目标

- 重做数据模型
- 新增首页后端事实源
- 恢复统一任务池
- 新增首页二级页或通用 Sheet
- 提前扩展到多家庭 SaaS 模型

## 当前验收重点

### 首页结构

- 继续固定 `逾期 / 繁育 / 健康 / 用药`
- 高密度时靠区块内结构优化，不靠静默截断

### 业务语义

- 繁育不是批量完成模型
- 批量健康完成必须创建真实 `health_record`
- 用药继续以 `medication_tasks` 为事实源

### 一致性

- pills 数字、WeekStrip 红点、首页实际可见内容一致
- 记录页返回后先本地承接再刷新
- suppression 与 latest token 继续防闪回

## 更新规则

- 路线变化时先更新本文件
- 如果变化涉及产品规则，再同步更新 `docs/design/02-features.md`
- 如果变化涉及工程边界，再同步更新 `docs/design/03-tech-stack.md` 与 `docs/design/04-implementation.md`
