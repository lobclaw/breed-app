# 记录表单统一验收清单

## 1. 目的

用于验收“按业务域统一表单内核”这次重构，重点确认以下风险没有回归：

- 同类型新增与编辑的字段、校验、提交行为一致
- 旧路由与旧 query 参数仍可打开正确页面
- 来源页提交反馈、返回承接、静默刷新不回归
- 用药仍保持“只统一创建，不开放编辑”的产品口径
- 繁育编辑页补齐后的额外安排读写与新增页一致

## 2. 前置准备

建议先准备以下测试数据，再开始逐项点击：

- 1 只在养种母：处于可做繁育记录状态
- 1 只公犬：用于配种记录
- 1 只幼犬：用于验证健康提醒间隔差异
- 至少 1 条疫苗记录
- 至少 1 条驱虫记录
- 至少 1 条疾病记录，且治疗状态不是 `已康复`
- 至少 1 条进行中的用药任务
- 至少 1 个发情中繁育周期
- 至少 1 条带额外安排的繁育记录

建议同时准备以下入口场景：

- 从犬只详情进入记录页
- 从首页卡片进入记录页
- 从详情页点击编辑进入编辑页
- 从批量卡/待办卡进入表单页

## 3. 健康域

### 3.1 疫苗新增

路径：
- `记录中心 -> 疫苗`
- `犬只详情 -> FAB -> 疫苗`

检查点：
- 犬只选择正常，多犬可选
- 疫苗类型、自定义疫苗、费用、备注都可填写
- `BFormOptions` 显示“标记为待办 + 日期 + 创建下次待办”
- 日期 chips 正常切换
- 勾选“标记为待办”后，按钮从“保存记录”切到“创建待办”
- 不勾选待办时提交会写入真实 `health_record`
- 勾选待办时提交只创建待办，不写健康记录
- 提交成功后返回来源页，来源页有弱成功反馈

### 3.2 疫苗编辑

路径：
- `健康记录详情 -> 编辑`
- 也可直接打开 `/pages/record/health-edit?id=<recordId>`

检查点：
- 页面标题为“编辑健康记录”
- 页面内显示只读的记录类型
- 字段值与新增页同口径，能正确回填
- 自定义疫苗仍可选、可新增、可删除
- 下次提醒日期能正确回填并可修改
- 提交后提示“已更新健康记录”，返回详情或来源页

### 3.3 驱虫新增

路径：
- `记录中心 -> 驱虫`
- `犬只详情 -> FAB -> 驱虫`

检查点：
- 驱虫类型切换后，药品候选随类型变化
- 自定义药品可新增、可删除
- `BFormOptions` 行为与疫苗页一致
- 从待办/批量卡进入时，`标记为待办` 区域应隐藏
- 从批量卡进入时，犬只与表单详情应自动预填

### 3.4 驱虫编辑

检查点：
- 与新增页字段结构一致
- 驱虫类型、药品、日期、费用、备注能正确回填
- 提交后不新增额外任务，只更新记录内容

### 3.5 疾病新增

路径：
- `记录中心 -> 疾病`
- `犬只详情 -> FAB -> 疾病`

检查点：
- 病症类型、自定义病症、严重程度、治疗状态都可填写
- 页面仍然隐藏“标记为待办”，不允许走待办模式
- 重复疾病校验仍生效
- 单犬且命中重复时，会提示跳转去编辑原记录
- 多犬或多条重复时，会给出 toast，不直接覆盖
- 提交成功且治疗状态不是 `已康复` 时，会弹“需要用药吗？”
- 选择“去创建用药”后，会带 `batchDogs + illnessRecordId` 跳到用药页

### 3.6 疾病编辑

检查点：
- 字段与新增页一致，不应出现新增有、编辑没有的字段
- 仍保留重复疾病兜底校验
- 治疗状态改为 `已康复` 后，可正常保存

## 4. 繁育域

### 4.1 主链新增页一致性

逐页检查以下页面：

- `breeding-heat`
- `breeding-follicle`
- `breeding-mating`
- `breeding-pregnancy`
- `breeding-prenatal`
- `breeding-prelabor`
- `breeding-termination`

每页通用检查点：

- 犬只锁定时，`BDogPicker` 为只读展示
- 日期 chips 都是 `今天 / 昨天 / 前天`
- 提交按钮三态一致
- 从任务入口进入时能预填 `dogId / dogName / cycleId / taskId`
- 提交后如果有 `taskId`，原任务会被处理完成
- 成功返回后来源页有承接反馈

### 4.2 主链字段专项

发情：
- 可录备注
- 可添加额外安排

卵泡检查：
- 左右卵泡数量/大小正常输入
- 检查结果可选

配种：
- 配种方式顺序仍为 `人工授精` 在前、`自然交配` 在后
- 默认选中 `人工授精`
- “第几脚”显示为系统自动计算，不提供手动编辑
- 预计孕检日、预计预产期自动计算
- 手动修改预产期后，保存结果正确

孕检：
- `确认怀孕` 开关正常
- 选择“是”后才显示幼崽数量
- 图片上传仍可用

临产：
- 低体温提示仍会出现
- 刨窝行为 toggle 正常

异常终止：
- 类型候选正常

### 4.3 发情观察

路径：
- `犬只详情 发情中 banner -> 发情观察`
- `繁育周期详情 -> 添加记录 -> 发情观察`
- 首页 `建议卵泡检查` 行弱动作 `观察`

检查点：
- 仅能挂到已有且状态为 `发情中` 的周期
- 外阴状态、分泌物状态、行为征兆、备注结构不变
- `分泌物状态` 仍是必填
- 不显示 `BExtraArrangementSection`
- 不联动费用
- 提交后只写 `heat_observation` 记录，不推进主链

### 4.4 繁育编辑

路径：
- `繁育记录详情 -> 编辑`
- 也可直接打开 `/pages/record/breeding-edit?id=<recordId>`

检查点：
- 各主链类型编辑页字段与对应新增页一致
- 记录类型只读展示正确
- 配种记录能回填种公、方式、手动预产期
- 孕检记录能回填图片和确认状态
- 临产记录能回填体温、刨窝行为、食欲变化、其他征兆
- 带额外安排的记录，编辑页能看到当前额外安排
- 修改额外安排后，关联的 `breeding_extra_arrangement` 任务同步更新
- 清空额外安排并保存后，关联任务应移除

注意：
- `heat_observation` 仍不是主链编辑口径重点，需单独验证其详情/删除限制，不要求变成通用主链编辑页

## 5. 用药域

### 5.1 用药创建

路径：
- `记录中心 -> 用药`
- `犬只详情 -> 开始用药`
- `疾病记录保存后 -> 去创建用药`

检查点：
- 犬只选择、多犬创建正常
- 药品名称、剂量、单位、方式、频率、持续天数、开始日期、费用、注意事项都可填写
- 方案库 Sheet 可打开、可回填
- 保存为方案仍可用
- 同药名重复检测仍生效
- 重复弹窗分为“将创建”和“已有同名任务”两段
- 重复犬默认不勾选覆盖
- 覆盖后语义仍是“取消旧任务 + 创建新任务”
- 单犬从疾病页跳来时，`illnessRecordId` 仍会带过去

### 5.2 用药详情

检查点：
- 仍只有查看、标记今日完成、取消用药
- 没有新增“编辑用药”入口
- 与本次表单统一重构不冲突

## 6. 路由兼容

### 6.1 健康编辑兼容

直接验证以下 URL 仍能打开：

- `/pages/record/health-edit?id=<recordId>`
- `/pages/record/health-edit?recordId=<recordId>`
- `/pages/record/health-edit?record_id=<recordId>`

### 6.2 用药创建兼容

直接验证以下 URL 仍能打开并预填：

- `/pages/record/health-medication?batchDogs=<encoded>`
- `/pages/record/health-medication?dogId=<dogId>&dogName=<dogName>`
- `/pages/record/health-medication?batchDogs=<encoded>&illnessRecordId=<id>`

### 6.3 繁育记录兼容

验证以下参数仍生效：

- `dogId / dogName`
- `cycleId`
- `taskId`
- `locked=true`

## 7. 返回承接

重点验证以下返回链路：

- 新增健康记录 -> 返回犬只详情
- 编辑健康记录 -> 返回健康详情
- 新增繁育记录 -> 返回繁育周期详情或来源页
- 编辑繁育记录 -> 返回繁育详情
- 疾病保存后跳用药 -> 用药保存后返回上一层

检查点：

- 提交成功后先走本地承接，再后台刷新
- 不出现强 success toast 驱动
- 提交按钮成功瞬态正常
- 犬只详情返回时不应整页骨架闪回

## 8. 建议执行顺序

建议按下面顺序点：

1. 健康域：疫苗新增 -> 疫苗编辑 -> 驱虫新增 -> 驱虫编辑 -> 疾病新增 -> 疾病编辑
2. 用药域：疾病跳用药 -> 用药创建 -> 用药详情
3. 繁育域：发情 -> 卵泡 -> 配种 -> 孕检 -> 产检 -> 临产 -> 异常终止
4. 繁育编辑：逐条验证与额外安排同步
5. 最后统一回归来源页承接与路由兼容
