# 测试清库

本文档用于在测试环境中把 App 数据恢复到接近初始状态，便于重新验证 local-first、离线写入和自动同步流程。

> 注意：以下操作会删除测试数据。执行前请确认当前连接的是测试云空间，不要在生产环境部署或调用 `dev-reset-service`。

## 1. 部署清库云对象

在 HBuilderX / UniCloud 面板中部署以下云对象：

```text
uniCloud-alipay/cloudfunctions/dev-reset-service
```

该云对象只用于测试环境手动清库，不参与业务链路。

## 2. 清空业务与同步数据

如果只需要重新建犬舍、重新录入犬只和业务数据，但保留当前登录账号，在 H5 控制台或临时调试入口执行：

```js
const reset = uniCloud.importObject('dev-reset-service')
await reset.resetTestData({ confirm: 'RESET_TEST_DATA' })
```

该操作会清空：

```text
families
dogs
breeding_cycles
litters
breeding_records
health_records
medication_tasks
tasks
expenses
incomes
sale_records
agents
dog_weights
medication_protocols
operation_logs
sync_mutations
attachment_deletions
```

## 3. 连账号数据一起清空

如果希望回到登录前状态，连 `uni-id` 账号数据也清掉，执行：

```js
const reset = uniCloud.importObject('dev-reset-service')
await reset.resetTestData({
  confirm: 'RESET_TEST_DATA_AND_AUTH',
  includeAuth: true,
})
```

该操作除业务集合外，还会清空：

```text
uni-id-users
uni-id-log
uni-id-device
opendb-device
opendb-verify-codes
opendb-tempdata
opendb-frv-logs
```

## 4. 清空本地缓存

H5 调试时，在浏览器 DevTools Console 执行：

```js
indexedDB.deleteDatabase('breed-app-local-first')
localStorage.clear()
location.reload()
```

这会同时清理旧 `kv` store 与 v2 `rows` store。

App 真机 / 模拟器调试时，最简单方式是卸载 App 或清除 App 数据。本地 SQLite 文件位于：

```text
_doc/breed_app_local_first.db
```

该 SQLite 数据库内保留旧 `local_kv` 表，并新增 v2 `local_rows` 表；清除 App 数据会一并删除。

## 5. 推荐重置顺序

1. 部署 `dev-reset-service`。
2. 调用 `resetTestData` 清空 UniCloud 测试数据。
3. 清空本地缓存。
4. 重新打开 App。
5. 如果保留账号，会进入创建犬舍流程；如果连账号一起清空，会先进入登录 / 注册流程。
