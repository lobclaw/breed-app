# 字段与页面映射

## 1. 文档目的

本文件不再逐字段展开成长清单，而是收敛成两类内容：

1. 每个业务域的“字段写入口”与“主要读取页面”
2. 当前已知审计结论与缺口

需要查单个字段时，应回到 `01-data-model.md` 作为模型事实源，再结合对应页面代码确认。

## 2. 域级映射

### 2.1 `dogs`

写入口：

- 新增犬只
- 编辑犬只
- 去向变更
- 生产向导中的幼崽创建
- 幼崽升级为种狗

主要读取页面：

- 犬只列表
- 犬只详情
- 窝详情
- 繁育周期详情
- 销售详情
- 首页卡片

联动字段：

- `name` 会冗余到繁育、财务、销售、首页任务上下文
- `role + disposition` 共同决定经营身份
- `origin_litter_id` 用于追溯幼崽来源
- `latest_weight` 由 `dog_weights` 最新记录同步

### 2.2 `breeding_cycles`

写入口：

- 发情/卵泡/配种记录自动创建
- 异常终止、手动关闭、生产、断奶等流程自动推进

主要读取页面：

- 繁育周期详情
- 犬只详情的繁育历史

关键联动：

- `status` 决定主流程状态
- `dam_* / sire_*` 是高频展示冗余字段

### 2.3 `breeding_records`

写入口：

- 发情
- 卵泡检查
- 配种
- 孕检
- 产检
- 临产
- 生产
- 异常终止

主要读取页面：

- 繁育周期详情时间线
- 记录详情页
- 首页流程卡跳转上下文

关键联动：

- `type` 决定 `details` 结构
- `cost` 可自动生成 `expenses`
- 生产记录会创建 `litters` 与幼崽 `dogs`

### 2.4 `litters`

写入口：

- 生产向导
- 窝内补录幼崽
- 断奶确认

主要读取页面：

- 窝详情
- 犬只详情中的来源窝
- 繁育周期详情

### 2.5 `health_records`

写入口：

- 疫苗表单
- 驱虫表单
- 疾病表单
- 首页批量健康完成

主要读取页面：

- 犬只详情健康记录
- 记录详情页
- 首页健康卡及疾病观察

关键联动：

- 疫苗/驱虫的显式待办创建
- 疾病唯一性校验
- 费用可联动生成支出

### 2.6 `medication_tasks`

写入口：

- 开始用药
- 重复药名覆盖重建
- 记录今日剂量
- 康复/停药/结束疗程

主要读取页面：

- 首页今日用药
- 用药详情
- 犬只详情当前状态

关键联动：

- 同狗同药名唯一
- 首页与详情页必须共用同一套状态判断

### 2.7 `tasks`

写入口：

- 健康记录显式创建待办
- 繁育主链推进
- 额外安排
- 用户手动待办
- 推迟、完成、跳过

主要读取页面：

- 首页
- 日期计数与 WeekStrip
- 部分来源页返回后的本地承接

关键联动：

- 批量卡 key 必须带健康子类型
- 批量完成后只按后端 `completedTasks` 局部移除

### 2.8 财务域

#### `expenses`

写入口：

- 手动记账
- 购入犬只
- 繁育记录带成本
- 生产记录带成本

主要读取页面：

- 财务列表
- 支出详情
- ROI/利润相关页面

#### `incomes`

写入口：

- 手动收入
- 销售完成
- 定金保留
- 领养收入

主要读取页面：

- 财务列表
- 收入详情
- 财务统计

#### `sale_records`

写入口：

- 底价、预定、成交、退款、定金取消

主要读取页面：

- 销售列表
- 销售详情
- 财务关联上下文

## 3. 当前审计结论

### 已收敛的口径

- 收入入口统一为 `pages/finance/expense-add.vue?type=income`
- `income-add.vue` 视为弃用
- 健康提醒改为建议型，默认不自动续链
- 繁育表单不再使用旧“下次提醒”字段
- 首页批量健康完成要求真实写入 `health_record`
- 疾病与用药在首页分区已经收口

### 仍需持续关注的区域

- 犬只详情“当前状态”排序与去重
- 首页局部承接时批量卡元数据同步
- 用药详情与首页漏服判断的一致性
- 繁育流程页的路由参数是否始终完整透传
- 冗余名称字段在改名后的同步完整性

## 4. 使用方式

当你需要修改某个字段时：

1. 先在 `01-data-model.md` 确认字段定义与业务规则
2. 再定位对应域的写入口页面或服务
3. 如果影响首页或聚合状态，补查 `02-features.md` 与 `docs/ROADMAP.md`
