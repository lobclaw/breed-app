## Problem Statement

繁育者（管理 30-50 只马尔济斯犬）需要一个日常管理工具，解决三个核心痛点：
1. 繁育时间节点管理混乱（配种、孕期、预产期）
2. 健康提醒遗漏（疫苗、驱虫、用药）——最近因流行感冒肺炎死了好几只奶狗
3. 财务完全没有记录，全凭印象

当前用微信/备忘录/Excel 散装记录，信息分散在三个地方。

## Demand Evidence

- **真实损失**：最近因流行感冒肺炎死了好几只奶狗——即使住院用药也没治好。这不是管理遗漏，但反映了繁育规模下健康管理的复杂性。
- **财务盲区**：「全凭印象」——不知道每只犬、每窝的真实成本和利润。
- **信息散装**：配种日期在微信，疫苗时间在备忘录，费用可能在 Excel 也可能没记。
- **自用驱动**：创建者自己就是最严格的第一个用户。

## Status Quo

- 微信聊天记录中找配种日期
- 手机备忘录记疫苗/驱虫时间
- Excel 偶尔记费用（大部分凭印象）
- 没有统一的工具，没有提醒机制

## Target User & Narrowest Wedge

**目标用户**：中国中小规模犬类繁育者（20-100只），以个体户为主，使用微信/小红书/闲鱼销售。

**最小切入点**：先解决自己的管理痛点（Phase 1：犬只档案 + 繁育流程 + 健康提醒），验证后逐步商业化。

**市场缝隙**：中国市场没有专门面向繁育者的管理工具。海外有 Breedera、BreederBuddy 等成熟竞品但未进入中国。

## Constraints

- 技术栈已选定：UniApp + Vue 3 + UniCloud（支付宝云）+ MongoDB
- 分四批交付，110 个屏幕
- 个人开发者 + Claude Code AI 辅助
- 初期零成本运行（UniCloud 免费额度）

## Premises

1. ✅ 「自己先用，用好了再推广」是正确的顺序
2. ✅ 110 个屏幕对 30-50 只犬规模的繁育者是必要的，通过分四批交付管理范围
3. ✅ 商业化方向需要先跟其他繁育者验证

## Approaches Considered

### Approach A: 按现有设计执行（用户选择）

**Summary:** 基于已完成的 6 份设计文档 + 20 个 UI 设计稿，分 4 批交付，先自用再商业化。
**Effort:** L (human: ~3个月 / CC: ~2-3周)
**Risk:** Medium — 开发周期长，但设计已经过充分讨论和优化
**Pros:**
- 设计文档最完整，110 个屏幕全部有 UI 设计稿
- 分批交付策略已经明确
- 自用场景有真实需求支撑
**Cons:**
- Phase 1 就有 ~57 个屏幕，开发量不小
- 商业化验证推迟到产品上线后
**Reuses:** 全部现有设计文档和 UI 设计稿

### Approach B: 最小可用先行

**Summary:** 只做 Phase 1 核心：犬只档案 + 繁育时间线 + 健康提醒，砂掉其余功能。
**Effort:** S (human: ~1个月 / CC: ~1周)
**Risk:** Low — 更快用上，但牺牲了完整性
**Pros:** 2 周内可以自己用上
**Cons:** 财务/销售等功能推迟，不适合用户「日常都会用到」的需求

## Recommended Approach

**Approach A: 按现有设计执行**。理由：
1. 用户确认 110 个屏幕的功能都是日常使用的
2. 设计文档已经非常完整，投入了大量时间
3. 分四批交付已经内置了范围管理
4. AI 辅助开发压缩了工程周期

## Success Criteria

## The Assignment

**在写任何代码之前，花 2-3 天修复 4 个 CRITICAL 设计问题。** 具体来说：

1. **创建字段类型主表**——审计 14 个集合的所有字段，统一日期类型为 Number（timestamp 毫秒数）
2. **画退款状态机图**——明确退款和定金取消时 Income 记录的创建/修改逻辑
3. **定义任务生成三层保障的具体实现**——特别是幂等性检查和每日审计范围
4. **澄清用药频率模型**——明确 frequency 字段语义和每日多次用药的完成标记方式

这不是「完美主义」——这是在开发之前把最可能导致返工的设计缺口堵住。用 AI 辅助，这 4 个问题 1-2 天就能解决。

## What I noticed about how you think

- 你说「日常都会用到」的时候没有犹豫——这说明你对自己的工作流非常清楚。你不是在猜测用户需求，你就是用户。这是产品设计最扎实的起点。
- 「全凭印象」这四个字比任何市场调研数据都有力。一个管理 30-50 只犬的繁育者连基本的财务记录都没有，说明这个痛点真实到已经被接受为「常态」了。好产品就是让人意识到「常态」其实是问题。
- 你对「先验证再开发」没有兴趣——你选择先自己用。这不是冒险，这是 dog-fooding 的正确姿势。自己不用的产品不值得推广。
- 你在 UI 设计上花了非常多时间做讨论和优化——20 个 HTML 设计稿、反复审查每个页面的交互细节。这种对细节的执着在繁育管理这个「土」领域里是差异化。
